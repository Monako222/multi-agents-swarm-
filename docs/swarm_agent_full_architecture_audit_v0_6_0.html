3. Порядок работы сервиса по шагам

  1. Фасад принимает query/files, создаёт thread_id, формирует входной state.
  2. Bootstrap очищает per-run каналы: errors/history/out_files/scratch/data, но сохраняет episodic memory.
  3. AgentNode сжимает history, строит runtime context, добавляет stable protocol prompt и вызывает LLM.
  4. Router всегда отправляет AIMessage с tool calls в tool-node, чтобы закрыть каждый tool_call_id.
  5. ToolExecutor исполняет non-routing tools параллельно, routing tools детерминированно, объединяет updates.
  6. Finalization нормализует ответ или восстанавливает fallback без технического текста для пользователя.

4. State, reducers и гарантия “ничего не потерять”

КаналСмыслReducer / правило
messagesLangChain message history.add_messages; старые сообщения удаляются через RemoveMessage.
contextНормализованный запрос, intent, subject, lang, tags.r_context, tags дедуплицируются и могут заменяться.
spaceОбщая память задачи и episodic_memory.r_space, notes дедуплицируются; episodic_memory не чистится на run init.
dataСтруктурированные findings: json, files, links.r_data, deep-merge + replace_value для reset.
workspaceЧерновик и финальный ответ.r_workspace; non-routing tools не могут писать final_answer.
errors/historyТехнический журнал и handoff-history.r_tail с bounded хвостом.
in_files/out_filesВходные/выходные файлы.r_unique + replace_list на новом run.
metricsLLM calls, tool calls, retries, prompt chars.r_metrics суммирует counters.
Почему нужен replace_value?

Для dict-каналов обычный patch означает deep-merge. Если на новом run передать {}, старые nested-ключи могли остаться. replace_value({}) явно говорит reducer: заменить значение целиком.

5. Сжатие контекста и message history

Алгоритм budget.compaction работает блоками: обычное сообщение или неразрывная пара AIMessage(tool_calls) + ToolMessage.... Неполные tool-блоки не попадают в активное окно, а уходят в episodic memory.

1. Разбивка на блоки

ToolMessage без AIMessage считается orphan. AIMessage с tool_calls валиден только при точном наборе ответов.

2. Выбор хвоста

Храним последние валидные блоки без разрыва протокола. Большая tool-пара сохраняется целиком.

3. Memory + RemoveMessage

Удалённые сообщения рендерятся в episodic_memory, а из state удаляются по id.

AI(tool_calls=[c1,c2]) + Tool(c1) + Tool(c2)  -> valid block
AI(tool_calls=[c1,c2]) + Tool(c1)             -> invalid block -> memory
Tool(c1) без предшествующего AI                -> orphan -> memory

6. Tools, parallel calls и routing barrier

Поддержка параллельных tool calls есть, но она не смешивает независимые side effects с маршрутизацией графа.

Правила executor

  • Non-routing tools до routing barrier могут выполняться параллельно.
  • finish приоритетнее transfer, потому что содержит готовый ответ.
  • Tool после выбранного routing tool не исполняется, но получает ToolMessage.
  • На loop-limit исполняется только finish; остальные вызовы закрываются warning-сообщениями.
  • Non-routing Command очищается от active_node, pending_transfer, counters, is_final и финального workspace.
  • Fan-out ограничен: max_tool_calls_per_turn, concurrency — max_parallel_tool_calls.
Практика для web/tools: если нужно найти два факта, лучше дать агенту batch-tool web_search_batch(queries=[...]). Если модель всё равно вернула два независимых calls, executor выполнит их параллельно.

8. SwarmService как фасад

async with SwarmService() as swarm:
    result = await swarm.ainvoke(
        "Проанализируй документы и дай краткий вывод",
        thread_id="user-123",
        files=[{"id": "doc1", "uri": "file:///tmp/a.pdf", "kind": "doc"}],
    )
    print(result.final_answer)

Фасад владеет LLMHub и HTTP pool. Он превращает graph crash в SwarmResult(ok=False) с безопасным пользовательским текстом. Технические детали остаются в snapshot.errors.

МетодЧто делает
ainvokeОсновной async вызов задачи, возвращает SwarmResult.
astreamПрокидывает LangGraph async stream events/state chunks.
astream_finalIterator, который отдаёт финальный ответ одним chunk.
aget_stateДостаёт checkpoint state для thread_id, если есть checkpointer.
acloseЗакрывает async HTTP pool.

9. Streaming

Сервис поддерживает LangGraph astream. По умолчанию лучше использовать stream_mode="values" для state snapshots. Token-level streaming нужно включать отдельным API-адаптером, чтобы отделить пользовательские токены от служебных tool/state events.

async for event in swarm.astream("Задача", stream_mode="updates"):
    handle(event)

async for text in swarm.astream_final("Задача"):
    yield text

10. Логи, LangSmith и Langfuse

Loguru

В коде используется logger.bind(...): thread_id, node, agent, tool, goto, retry. По умолчанию сервис перенастраивает Loguru на INFO, чтобы не шумел дефолтный DEBUG sink. Если приложение само управляет логами, поставь SWARM_LOG_CONFIGURE=false. Для JSON-логов:

SWARM_LOG_CONFIGURE=true
SWARM_LOG_LEVEL=INFO
SWARM_LOG_JSON=true

LangSmith

LangGraph/LangChain умеют прокидывать tracing metadata/tags через RunnableConfig. Практический план:

config={
    "tags": ["prod", "swarm", "v0.6"],
    "metadata": {"tenant_id": "...", "request_id": "..."},
}
result = await swarm.ainvoke("...", config=config)

Langfuse

Langfuse подключается через LangChain callbacks. В будущем фасад можно расширить параметром callbacks или прокидывать их через config.

from langfuse.langchain import CallbackHandler
handler = CallbackHandler()
await swarm.ainvoke("...", config={"callbacks": [handler]})
PII: перед production tracing нужны anonymizers/redaction rules, чтобы не логировать секреты, персональные данные и содержимое приватных документов без разрешения.

11. План масштабирования

Tools по категориям

web_search_batch, fetch_url, parse_pdf, extract_tables, analyze_image, transcribe_audio, extract_keyframes. Все IO-tools должны иметь timeout, retry policy, size limits и allowlist.

MCP

Добавить tools/mcp adapter: загрузка server manifest, allowlist tools, schemas, sandbox permissions, human approval для write/delete/network-heavy actions.

Remote agents / A2A

Расширить PendingTransfer(protocol=A2A): endpoint_uri, agent_card_uri, auth, heartbeat, timeout, result schema, idempotency key.

Persistence

Подключить production checkpointer: Postgres/Redis/S3-compatible storage, retention policy, thread isolation, encryption at rest.

Evaluation

Regression prompts, handoff correctness, tool schema conformance, compaction invariants, source citation correctness, cost/latency benchmarks.

Security

Prompt-injection scanner для tool outputs, per-tool permissions, audit log, quota/rate limits, redaction before logs/traces.

Как добавить новый tool category

src/swarm_agent/tools/web/__init__.py

def build_web_tools(caller_name: str) -> tuple[BaseTool, ...]:
    return (web_search_batch_tool,)

# затем в AgentSpec:
AgentSpec(name="web", ..., tools=("web",))

13. Production checklist

Перед коммитом
  • Нет секретов в коде, README, tests, .env.example.
  • python -m compileall -q src tests проходит.
  • PYTHONPATH=src pytest -q -rs проходит.

  • CI: tests, static audit, dependency lock, vulnerability scan.
